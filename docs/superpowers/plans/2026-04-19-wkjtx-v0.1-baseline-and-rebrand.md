# WKjTX v0.1 — Baseline build and rebrand — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a working Windows build of JTDX rc160 rebranded as WKjTX that launches, shows the expected UI, and stores config under `%LOCALAPPDATA%\WKjTX\`. No feature additions yet — this is the foundation.

**Architecture:** Fork JTDX rc160 source, vendored inside this repository under `jtdx-source/`. Build using MSYS2 + Qt 5.15 + gfortran + JTDX-Hamlib. Apply a pure rebrand pass (strings, icons, paths, installer) without touching decoder logic. Commit the vendored source as a baseline reference, then commit the rebrand as a single reviewable diff.

**Tech Stack:** C++14/17, Qt 5.15, Fortran, CMake, Ninja, NSIS, MSYS2/MinGW-w64, Git.

**Scope boundaries for v0.1:**
- In: toolchain install, source clone, first clean build, rebrand pass, installer rebrand, launch verification.
- Out: all features (profiles, auto-call, qrz.com, log routing) — those are Plan 2, 3, 4.

### Task 0.6: First vanilla build of JTDX

Goal: compile JTDX as-is, without any modifications. This validates the toolchain and produces a known-good reference binary for regression comparison later.

**Files:**
- Create: `G:\Claude Local\WKjTX\build-artifacts\jtdx-vanilla\`

- [ ] **Step 1: Configure CMake**

```bash
cd /g/Claude\ Local/WKjTX/jtdx-source
mkdir -p build-vanilla
cd build-vanilla
cmake -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_PREFIX_PATH="/g/Claude Local/WKjTX/build-artifacts/hamlib" \
  -DCMAKE_INSTALL_PREFIX="/g/Claude Local/WKjTX/build-artifacts/jtdx-vanilla" \
  ..
```

Expected: CMake prints `-- Configuring done` and `-- Generating done`. If it fails with "Could not find Qt5" or "Could not find FFTW", the MSYS2 environment is not active (you're in the wrong shell) — reopen `MSYS2 MINGW64` and rerun.

- [ ] **Step 2: Compile**

```bash
ninja
```

Expected: build completes in 10–30 minutes depending on CPU. If it fails partway, read the error carefully. Common causes and fixes:

| Error | Fix |
|---|---|
| `hamlib/rig.h: No such file or directory` | `-DCMAKE_PREFIX_PATH` wrong in step 1. Re-run with corrected path. |
| `undefined reference to 'png_*'` | libpng missing. `pacman -S mingw-w64-x86_64-libpng` and rerun `ninja`. |
| `fatal error: QtWidgets: No such file or directory` | Qt5 not found. Re-check Task 0.2 Step 2. |
| `character-length argument must be...` (Fortran) | Known rc160 issue. Apply workaround: `export FFLAGS="-fallow-argument-mismatch"` and reconfigure (delete `CMakeCache.txt` first, rerun Step 1). |

Record any fixes you apply — they go into `INSTALL-WKjTX.md` in Task 1.10.

- [ ] **Step 3: Launch the binary**

```bash
./jtdx
```

Expected: JTDX window opens. Title bar says "JTDX vX.Y.Z". Close it.

- [ ] **Step 4: Capture reference decoder output for regression**

Download a reference test `.wav` file from the WSJT-X test suite or use one the user has:

# Building WKjTX from source (Windows 10/11 64-bit)

## 1. Prerequisites

- Windows 10 or Windows 11, 64-bit.
- 20 GB free disk space.
- Administrator account (for MSYS2 installer prompts).

## 2. Install MSYS2

Download the MSYS2 installer from https://www.msys2.org/ and run
it with default settings. Install path: `C:\msys64`.

After install, open `MSYS2 MINGW64` from the Start menu (not
`MSYS2 MSYS`) and run:

```bash
pacman -Syu --noconfirm
```

The shell will close. Reopen `MSYS2 MINGW64` and run:

```bash
pacman -Su --noconfirm
```

## 3. Install build dependencies

In `MSYS2 MINGW64`:

```bash
pacman -S --needed --noconfirm \
  mingw-w64-x86_64-gcc \
  mingw-w64-x86_64-gcc-fortran \
  mingw-w64-x86_64-cmake \
  mingw-w64-x86_64-ninja \
  mingw-w64-x86_64-pkgconf \
  mingw-w64-x86_64-make \
  mingw-w64-x86_64-qt5-base \
  mingw-w64-x86_64-qt5-tools \
  mingw-w64-x86_64-qt5-multimedia \
  mingw-w64-x86_64-qt5-serialport \
  mingw-w64-x86_64-qt5-translations \
  mingw-w64-x86_64-qwt-qt5 \
  mingw-w64-x86_64-fftw \
  mingw-w64-x86_64-boost \
  mingw-w64-x86_64-libusb \
  mingw-w64-x86_64-nsis \
  git \
  patch \
  unzip
```

## 4. Clone WKjTX

```bash
cd /g
git clone <your-github-url>/wkjtx.git wkjtx
cd wkjtx
```

## 5. Build JTDX-Hamlib

```bash
cd third-party/jtdxhamlib
./bootstrap
mkdir -p build && cd build
../configure --prefix=$PWD/../../build-artifacts/hamlib \
  --disable-shared --enable-static \
  --without-cxx-binding --disable-winradio
make -j$(nproc)
make install
```

## 6. Build WKjTX

```bash
cd /g/wkjtx/jtdx-source
mkdir build-wkjtx && cd build-wkjtx
cmake -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_PREFIX_PATH="/g/wkjtx/build-artifacts/hamlib" \
  -DCMAKE_INSTALL_PREFIX="/g/wkjtx/build-artifacts/wkjtx" \
  ..
ninja
```

Expected build time: 10–30 minutes.

## 7. Launch

```bash
./wkjtx.exe
```

## 8. Create installer (optional)

```bash
cpack -G NSIS
```

Produces `wkjtx-setup-0.1.0.exe` in the build directory.

## Known build gotchas

- **`character-length argument` Fortran error**: add
  `FFLAGS="-fallow-argument-mismatch"` to the cmake invocation.
- **`undefined reference to png_*`**: install
  `mingw-w64-x86_64-libpng`.
- **`Qt5 not found`**: ensure you are in `MSYS2 MINGW64`, not
  `MSYS2 MSYS` or `MSYS2 UCRT64`.
```

Adjust the clone URL and /g/wkjtx path once you know them.
