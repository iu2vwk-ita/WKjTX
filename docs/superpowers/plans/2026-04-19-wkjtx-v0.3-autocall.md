# WKjTX v0.3 ‚Äî Auto-call ‚Äî Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port the auto-call feature from FT8 Card Pro (PySide6) natively into WKjTX as a new Impostazioni tab with seven trigger categories, hardcoded safeguards, and a flashing top-bar badge. Each profile owns its own auto-call config.

**Architecture:** Fill in `wkjtx::AutoCall::onDecode()` (skeleton from v0.1.0) with a detector-pipeline pattern. Hook into the decode display path at `mainwindow.cpp:3857` (`displayDecodedText`). Add a new tab to `Configuration.ui` with the seven category checkboxes + 5 alert slots. Add the flashing badge widget to the MainWindow status bar.

**Tech Stack:** Qt 5.15, reuse of JTDX's existing CTY.DAT lookup and ADIF parser (`logbook/adif.cpp`).

**Prerequisites:** Plan 2 (v0.2.0) complete. ProfileManager in place so each profile can own its own AutoCallConfig.
