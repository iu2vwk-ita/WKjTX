# WKjTX v0.2 — Profile system — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a 5-slot radio profile quick-switch system (toolbar + F1–F5 hotkeys) to WKjTX v0.1.0 so the user can flip between up to five complete configurations — CAT, audio, UDP ports, log path, macros, auto-call config — with one click.

**Architecture:** Extend JTDX's existing `Configuration` class with per-slot INI I/O, implement `wkjtx::ProfileManager` (skeleton already landed in v0.1.0 at `jtdx-source/wkjtx/ProfileManager.{hpp,cpp}`), insert a permanent `QToolBar` in `MainWindow`, register F1..F5 `QShortcut`s, and wire `ProfileManager::switchToSlot()` into the existing CAT/audio/UDP lifecycle.

**Tech Stack:** C++14/17, Qt 5.15, existing JTDX `Configuration` class as the per-slot state container.

**Prerequisites:** Plan 1 (v0.1.0) complete. Working Windows build of WKjTX.

- [ ] **Step 3: Implement `ProfileManager::saveSlot`**

Replace the stub in `jtdx-source/wkjtx/ProfileManager.cpp` `saveSlot` (currently returns false) with a working implementation that writes `name`, `colorTag`, `hotkey`, and `logPath` to `slot<N>.ini`. Mirrors `writeProfileIni()` which is already wired.

```cpp
bool ProfileManager::saveSlot (int slotIndex)
{
  if (slotIndex < 1 || slotIndex > kMaxSlots) return false;
  ensureProfilesDirExists ();
  QString const path = QStringLiteral ("%1/slot%2.ini").arg (profilesDir_).arg (slotIndex);
  QSettings ini {path, QSettings::IniFormat};
  return writeProfileIni (slots_[slotIndex - 1], ini);
}
```
